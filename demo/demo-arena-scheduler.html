i.
What Mode A buys you
Steady 60 fps with a flat heap. Arena packs particles into Float32Arrays so the physics inner loop is monomorphic. Scheduler drives systems via MessageChannel at Priority.Normal; the tap handler runs at UserInput.
ii.
When Arena goes off
Particles become Array<{x,y,vx,vy}>. Property reads are polymorphic; .splice is O(n). Frame cost climbs and the heap line walks upward as dead-particle objects pile up between GCs.
iii.
When Scheduler goes off
Systems run via setTimeout(0). The browser clamps it to ~1 ms minimum; tap latency jumps. Click the tap test under load and watch the sparkline climb past 16 ms — a dropped frame just for the input handler.
